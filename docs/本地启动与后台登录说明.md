# Drama Pulse 本地启动与后台登录说明

## 0. 适用范围

本文档用于说明当前仓库在本地如何启动服务端、管理后台和 Android 客户端，以及后台令牌、端口占用和局域网联调的常见问题。

## 1. 服务端读取哪个环境文件

当前服务端会按以下顺序读取环境变量：

1. 项目根目录 `.env`
2. `server/.env`

如果两个文件里有同名字段，`server/.env` 优先级更高。

## 2. 当前本地后台令牌与端口

当前默认后台登录令牌为：

`drama-pulse-admin-token-2024`

来源：

- 根目录 `.env`
- `server/.env`

当前默认端口：

- 服务端：`8787`
- 管理后台前端开发服务：`5173`

## 3. 推荐启动顺序

### 3.0 首次安装依赖

服务端与后台使用 `pnpm`，Android 使用 Gradle。

```bash
cd /Users/a0000/Desktop/项目文件/drama-pulse/server
pnpm install

cd /Users/a0000/Desktop/项目文件/drama-pulse/admin-web
pnpm install
```

### 3.1 启动服务端

```bash
cd /Users/a0000/Desktop/项目文件/drama-pulse/server
pnpm dev
```

默认监听：

- `http://localhost:8787`
- `http://127.0.0.1:8787`
- `http://<局域网IP>:8787`

### 3.2 启动 Web 管理后台

```bash
cd /Users/a0000/Desktop/项目文件/drama-pulse/admin-web
pnpm dev
```

默认打开：

- `http://localhost:5173`

### 3.3 启动 Android 客户端

推荐直接用 Android Studio 打开 `android-app/`。

如果要用命令行构建：

```bash
cd /Users/a0000/Desktop/项目文件/drama-pulse/android-app
./gradlew assembleDebug
```

如果是真机联调，需要确认客户端访问的是服务端所在电脑的局域网 IP，而不是模拟器专用地址。

## 4. 后台登录方式

打开登录页后，在“后台令牌”输入框中输入：

`drama-pulse-admin-token-2024`

后台会自动带上：

`Authorization: Bearer <ADMIN_TOKEN>`

然后请求：

- `/admin/dramas`
- `/admin/highlights`
- `/admin/branch-tasks`

如果前端接口地址没有改过，默认会请求：

- `http://localhost:8787`

## 5. 401 的常见原因

### 5.1 Token 不对

如果输入的 token 不等于服务端 `ADMIN_TOKEN`，会返回 `401 Unauthorized`。

### 5.2 服务端没有重启

如果你刚修改过 `.env`，但服务端没重启，旧进程仍然会用旧配置。

### 5.3 浏览器里残留了错误 token

后台 token 保存在浏览器 `localStorage`：

- key: `drama-pulse-admin-token`

如果之前输错过，可以：

1. 清 `localStorage`
2. 或直接重新输入正确 token

### 5.4 服务端端口被占用

如果启动服务端时看到 `EADDRINUSE: address already in use 0.0.0.0:8787`，说明 `8787` 端口已经被其他进程占用。

可以先查看占用：

```bash
lsof -i :8787
```

确认后选择：

- 结束旧进程
- 或修改 `.env` / `server/.env` 中的 `PORT`

## 6. 建议的本地开发配置

### 根目录 `.env`

用于团队统一记录常用本地配置，例如：

- `ADMIN_TOKEN`
- `HOST`
- `PORT`

### `server/.env`

用于服务端本地覆盖配置。

### `admin-web/.env`

用于管理后台前端接口地址配置。

默认示例：

- `VITE_API_BASE_URL=http://localhost:8787`

## 7. 局域网联调建议

如果需要 Android 真机访问本地服务端，建议按下面方式确认：

1. 服务端监听 `0.0.0.0`
2. 手机和电脑处于同一局域网
3. 管理后台与手机都改用电脑的局域网 IP 访问服务端
4. 如果切换了端口或 IP，记得同步更新 `admin-web/.env` 和 Android 端请求地址
