# Drama Pulse 本地启动与后台登录说明

## 1. 服务端读取哪个环境文件

当前服务端会按以下顺序读取环境变量：

1. 项目根目录 `.env`
2. `server/.env`

如果两个文件里有同名字段，`server/.env` 优先级更高。

## 2. 当前本地后台令牌

当前默认后台登录令牌为：

`drama-pulse-admin-token-2024`

来源：

- 根目录 `.env`
- `server/.env`

## 3. 启动顺序

### 3.1 启动服务端

```bash
cd /Users/a0000/Desktop/项目文件/drama-pulse/server
pnpm dev
```

默认监听：

- `http://localhost:8787`

### 3.2 启动 Web 管理后台

```bash
cd /Users/a0000/Desktop/项目文件/drama-pulse/admin-web
pnpm dev
```

默认打开：

- `http://localhost:5173`

## 4. 后台登录方式

打开登录页后，在“后台令牌”输入框中输入：

`drama-pulse-admin-token-2024`

后台会自动带上：

`Authorization: Bearer <ADMIN_TOKEN>`

然后请求：

- `/admin/dramas`
- `/admin/highlights`
- `/admin/branch-tasks`

## 5. 401 的常见原因

### 5.1 Token 不对

如果输入的 token 不等于服务端 `ADMIN_TOKEN`，会返回 `401 Unauthorized`。

### 5.2 服务端没有重启

如果你刚修改过 `.env`，但服务端没重启，旧进程仍然会用旧配置。

### 5.3 浏览器里残留了错误 token

后台 token 保存在浏览器 `localStorage`：

- key: `drama-pulse-admin-token`

如果之前输错过，可以：

1. 清 `localStorage`
2. 或直接重新输入正确 token

## 6. 建议的本地开发配置

### 根目录 `.env`

用于团队统一记录常用本地配置，例如：

- `ADMIN_TOKEN`
- `HOST`
- `PORT`

### `server/.env`

用于服务端本地覆盖配置。

### `admin-web/.env`

用于管理后台前端接口地址配置。
