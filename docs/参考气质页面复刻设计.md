# Drama Pulse 参考气质页面复刻设计

## 1. 文档目的

本文档用于基于以下三个参考页面，抽取适合 Drama Pulse 项目的 Android 端页面风格与结构：

- 选剧页参考
- 播放页参考
- 个人主页参考

目标不是 1:1 照搬，而是：

- 保留参考产品的短剧平台气质
- 去掉与本项目无关的商城、赚钱、复杂运营位等干扰项
- 收敛成更适合 `短剧播放 + 高光互动 + 尾集分支` 的版本
- 为后续 Android UI 实现提供稳定的页面结构依据

## 2. 总体设计结论

三个页面统一继承以下气质：

- 高信息密度，但不杂乱
- 大封面、大视频、大头像区域优先
- 圆角卡片 + 浅背景 + 轻阴影
- 首页与个人页偏亮色、偏轻快
- 播放页偏沉浸式、偏深色覆盖层
- 底部导航固定三栏：`选剧`、`播放`、`我的`

需要明确舍弃的部分：

- 商城
- 赚钱入口
- 过多运营 Banner
- 多余任务入口
- 与本项目无关的搜索副能力
- 复杂社交动态

## 3. 视觉语言抽取

### 3.1 页面气质

参考图的核心气质不是“高级感”，而是：

- `内容密集但上手快`
- `短剧平台的效率感`
- `强封面驱动`
- `操作入口大且直接`

适合本项目的收口方式：

- 保留大图卡片和强封面风格
- 减少无关入口，让每屏只围绕一个核心任务
- 保留圆角、留白、轻色块氛围
- 用更稳定的组件体系替代参考图里较杂的商业化入口

### 3.2 色彩方向

建议整体以浅色为主，播放页局部深色：

- 页面背景：`#F7F8FA`
- 主卡片背景：`#FFFFFF`
- 主文本：`#111111`
- 次文本：`#8A8F98`
- 分隔线：`#E9ECF1`
- 强调色：`#FF7A2F`
- 活跃橙点：`#FF8C2E`
- 柔和高亮：`#FFF3E8`
- 播放页遮罩：`rgba(0,0,0,0.32)`

### 3.3 圆角与间距

建议页面统一：

- 页面左右安全边距：`16dp`
- 卡片圆角：`20dp`
- 小标签圆角：`10dp`
- 组件纵向间距：`12dp`
- 模块间距：`20dp`
- 底部导航高度：`64dp`

### 3.4 字体层级

- 页面主标题：`30sp / Bold`
- 模块标题：`20sp / Bold`
- 卡片主标题：`17sp / Semibold`
- 普通正文：`14sp / Regular`
- 标签文案：`11-12sp / Medium`
- 次级说明：`12sp / Regular`

## 4. 页面收口策略

### 4.1 选剧页

参考图保留点：

- 顶部搜索条气质
- 分类横向导航
- 快捷功能入口卡
- 双列剧卡瀑布感
- 热度数字压在封面图上的形式

对本项目的收口：

- 搜索条保留，但只做轻搜索入口，不接复杂识剧
- 分类只保留与短剧相关的少量标签
- 快捷入口只保留 3 个：`分类筛选`、`热度排行`、`最新上架`
- 剧卡只展示和项目有关的信息：封面、标题、热度、标签
- 不做商城/听书/电影等跨内容入口

### 4.2 播放页

参考图保留点：

- 全屏竖版沉浸式播放
- 顶部极简工具位
- 右侧互动操作带
- 视频上叠层信息
- 底部关键信息卡片

对本项目的收口：

- 不直接复刻短视频“点赞评论分享”逻辑
- 右侧操作区改成与本项目更相关的：
  - 高光热度
  - 收藏
  - 评论入口
  - 分享
- 中下部信息区保留“剧名 + 集数 + 标签 + 简介”
- 底部大卡片改成“完整短剧入口 / 选集入口 / 尾集分支入口”
- 主播放页必须为后续高光互动预留浮层区

### 4.3 个人主页

参考图保留点：

- 顶部头像 + 用户名 + 简单状态
- 一排统计数字
- 横向快捷功能卡
- 历史/收藏/点赞 tab
- 剧目浏览历史网格

对本项目的收口：

- 去掉订单、消息、领钱等电商化信息
- 保留 3 个核心二级区：
  - 历史
  - 收藏
  - 我的分支
- 用户状态只保留本地昵称和设备身份，不引入复杂认证
- 网格内容以“继续观看历史”和“已生成分支结果”为主

## 5. 三个页面的最终结构定义

## 5.1 选剧页

### 页面目标

帮助用户快速选中剧并进入播放，不承担复杂内容发现。

### 结构

1. 顶部状态区
2. 搜索条
3. 分类导航
4. 快捷功能卡行
5. 推荐剧双列卡片区
6. 底部三栏导航

### 必保留元素

- 搜索框
- 分类标签
- 双列剧卡
- 热度数字
- 剧名
- 2 个以内标签
- 当前导航高亮

### 去掉元素

- 无关内容类型入口
- 多层嵌套标签
- 过多榜单入口
- 二级商业引导位

## 5.2 播放页

### 页面目标

让用户沉浸式看剧，同时为高光互动和尾集分支预留稳定舞台。

### 结构

1. 全屏视频层
2. 顶部透明操作栏
3. 右侧竖向操作带
4. 中下部剧情信息层
5. 底部主操作卡
6. 底部三栏导航

### 必保留元素

- 视频主视觉
- 返回/菜单
- 搜索或更多入口其一
- 剧名
- 集数
- 标签
- 简介摘要
- 完整短剧入口
- 高光浮层预留位

### 去掉元素

- 过多社交数字
- 与本项目无关的电商挂件
- 多层弹窗式推荐

## 5.3 个人主页

### 页面目标

承接用户自己的观看记录、收藏和分支内容，不做复杂社区主页。

### 结构

1. 顶部浅色背景区
2. 头像 + 昵称 + 简短身份说明
3. 统计信息行
4. 功能入口行
5. 一级 Tab：历史 / 收藏 / 我的分支
6. 二级筛选条
7. 内容网格
8. 底部三栏导航

### 必保留元素

- 头像
- 昵称
- 统计数字
- 历史记录
- 收藏记录
- 分支记录
- 剧目卡片网格

### 去掉元素

- 订单
- 消息中心
- 收益体系
- 审核状态、任务体系等无关元素

## 6. 底部导航统一规则

底部导航统一为三个入口：

- `选剧`
- `播放`
- `我的`

设计要求：

- 固定白底
- 当前页为深色高亮文字
- 图标线性简洁
- 中间不做凸起按钮
- 页面切换不应出现复杂动画

## 7. 组件复用建议

三页建议复用的基础组件：

- `TopSearchBar`
- `CategoryTabs`
- `QuickActionCard`
- `DramaPosterCard`
- `BottomNavBar`
- `ProfileStatItem`
- `ProfileSectionTabs`
- `HistoryDramaGridItem`
- `PlayerSideActionRail`
- `PlayerInfoPanel`
- `BottomActionCard`

## 8. 页面差异化重点

### 选剧页重点

- 强调“找剧效率”
- 重点是封面浏览和快速进入

### 播放页重点

- 强调“沉浸观看”
- 后续高光互动要能自然叠加

### 个人主页重点

- 强调“记录沉淀”
- 不做复杂个人社交表达

## 9. 对 Android 实现的建议

如果后续要落地到 Compose：

- 先实现结构，不先追细节动画
- 优先保证三个页面的块级布局稳定
- 图像资源先用占位图和剧封面替代
- 底部导航与卡片体系先统一
- 播放页必须预留高光互动叠层

## 10. 结论

这三页可以复刻，而且适合复刻成更收敛的版本。

最终不建议照搬参考图，而建议采用：

`参考平台气质 + 我们项目主链路收口 + 去商业干扰项`

这样既能保留用户熟悉的短剧平台体验，也能让 Drama Pulse 的高光互动和尾集分支更自然地挂上去。
