# 高光资产生成与接入方案

## 1. 文档目标

本文档单独收口 Drama Pulse 高光组件的视觉资产生产方式、交付格式、目录组织和 Android 接入策略。

它解决的不是“高光应该长什么样”这个问题，而是更具体的执行问题：

- 现阶段是否还能继续只靠代码做效果
- 如果要更逼近概念图，应补哪些真实素材
- 能否用生成式方法快速做出第一版资产包
- 这些资产如何拆分、命名、管理和接入

统一结论：

- 现有客户端代码只能做出“有功能的原型感”
- 如果要逼近概念图观感，必须补真实视觉资产
- 最省事的落地方式不是从零手工绘制整套动画，而是采用
  `生成式主资产 + 轻量运行时编排`

## 2. 当前现状判断

截至当前版本，项目里真正可用的高光资产非常少。

当前已有内容主要是：

- 五张概念参考图，位于
  [docs/assets/highlight-concepts](/Users/a0000/Desktop/项目文件/drama-pulse/docs/assets/highlight-concepts)
- Android 端现有高光效果主要仍由
  [HighlightOverlay.kt](/Users/a0000/Desktop/项目文件/drama-pulse/android-app/app/src/main/java/com/dramapulse/app/ui/overlay/HighlightOverlay.kt)
  通过 `Text + Canvas + blur + 渐变` 拼接实现

这意味着：

- 可以快速验证交互逻辑
- 可以验证大致的布局位置和节奏
- 但无法稳定做出“像成品”的质感

## 3. 为什么纯代码方案不够

纯 Compose / Canvas 方案有它的价值，但它更适合做这些事：

- 位置控制
- 节奏控制
- 透明度和缩放控制
- 多次点击递进
- 边缘氛围的轻量调度

它不擅长稳定复刻这些东西：

- 有设计感的主字效质感
- 裂纹、火线、余烬、柔光、粒子细节
- 图形边缘的“脏感”“糙感”“爆裂感”
- 风格统一的按钮本体与峰值爆发层

结论不是“代码没用”，而是：

- 代码负责编排
- 真实素材负责质感

## 4. 推荐落地路线

最适合当前项目的路线是：

`生成式资产生产 -> 资产拆包 -> Android 轻量接入 -> 真机调试 -> 局部重做`

也就是：

1. 先用生成式方式快速产出第一版资产母版
2. 把资产拆成若干可复用的小层
3. Android 端只负责时机、点击、漂浮、退出
4. 真机调试后，哪一类不够好，再局部重做

这条路线的优点：

- 省去从零手工画整套资源的成本
- 比纯代码方案更接近概念图
- 比直接上复杂逐帧动画更容易迭代
- 便于后续逐类替换成更高级资源

## 5. 资产生产策略

### 5.1 不直接做“大整包动画”

第一版不要追求每类高光都做成一整段完整逐帧动画。

应该拆成 4 类资产：

- 触发按钮资产
- 主视觉资产
- 边缘氛围资产
- 峰值爆发资产

原因：

- 更容易复用
- 更容易替换某一层
- Android 接入更轻
- 多次点击时可以自由组合，而不是每次重播整段动画

### 5.2 生成式优先，手工精修作为补充

推荐优先顺序：

1. 先用生成式工具产出主视觉和边缘元素
2. 再由代码编排它们的入场、点击、漂浮和淡出
3. 若某一类效果仍然不够，再补设计师精修或动效师资产

这意味着第一版资产生产不依赖完整设计团队，也能推进。

## 6. 五类高光的资产拆分标准

所有类型统一拆成以下结构。

### 6.1 触发按钮层

用途：

- 默认态露出
- 引导用户点击

建议交付：

- `button_idle`
- `button_pressed`
- `button_glow`

要求：

- 透明背景
- 支持深色视频背景叠加
- 默认尺寸适配移动端中下区域

### 6.2 主视觉层

用途：

- 用户点击后的中心主表现

示例：

- `feel_good`：爽字主图
- `funny`：哈 / 哈哈哈主图
- `reversal`：卧槽 / WOC 主图
- `conflict`：火焰能量主图
- `sweet`：温情爱心主图

### 6.3 边缘氛围层

用途：

- 让点击后不只剩一个中心按钮
- 形成空间感和气氛

示例：

- `feel_good`：少量爽字、碎光、亮点
- `funny`：小哈字、上浮粒子
- `reversal`：小卧槽、裂片、冷白闪光
- `conflict`：火线、余烬、热浪
- `sweet`：小爱心、暖光、柔雾

### 6.4 峰值爆发层

用途：

- 承担点击后最强的那一下视觉反馈

要求：

- 时长短
- 冲击强
- 不能遮满中间主视频

### 6.4 峰值爆发层

用途：

- 承担点击后最强的那一下视觉反馈

要求：

- 时长短
- 冲击强
- 不能遮满中间主视频

### 6.5 `reversal` 的实际素材搭配

`reversal` 这一类不再按单图使用，而是按分层组合：

- `button_idle`：`highlight_reversal_button_primary_user_v1`，默认触发按钮
- `energy_ring`：`highlight_reversal_energy_ring`，常驻底层能量环
- `burst_core`：`highlight_reversal_crack_cluster`，点击瞬间爆发层
- `edge_echo`：`highlight_reversal_edge_woc_v1` / `highlight_reversal_edge_woc_v2`，边缘回声层

默认态只保留按钮 + 弱能量环；点击态再叠加裂片和边缘小字。不要把四层同时常驻，否则画面会变脏。

### 6.6 `reversal` 首轮落地后的复用经验

这次真正跑通 `reversal` 后，有几条经验需要固定下来，后面做别的高光时直接复用：

- 素材不要一开始就追求“大而全”的完整动效包，优先拆出可独立替换的小层
- 最小可用组合优先级是：
  `中心按钮 -> 中心主字/主图 -> 边缘回声层 -> 点击爆发层`
- 中心按钮和主字不是一类素材：
  按钮负责引导点击，主字负责点击后的情绪爆发，最好分文件维护
- 边缘素材不要只准备 1 张：
  至少准备 2 到 3 个同风格小素材，运行时随机取样，画面才不会太死
- 透明背景是硬要求，避免 Android 侧为了抠底再做额外处理
- 边缘素材本身最好就是“正角度可用”的，不依赖运行时大角度翻转

针对 `reversal`，当前这套最小资产包已经验证可用：

- `highlight_reversal_button_primary_user_v1`
- `highlight_reversal_core_woc`
- `highlight_reversal_core_wocao`
- `highlight_reversal_crack_cluster`
- `highlight_reversal_edge_woc_v1`
- `highlight_reversal_edge_woc_v2`

补充原则：

- “按钮位置”这套空间策略可以复用到后续强互动高光，优先沿用当前 `reversal` 的中下触发位
- 但“按钮之上的视觉模板”不要机械复用到所有类型
- 比如 `reversal` 适合骤停后爆发，`sweet`、`funny`、`feel_good` 只建议复用按钮宿主与调参方式，不直接复用裂片、边缘回声和冷白爆闪

### 6.7 `funny` 的实际素材搭配

`funny` 当前也已经从概念图推进到真实可用素材包，第一版实际保留并接入的是：

- `button_idle`：`highlight_funny_button_ha`，默认触发按钮
- `hero_core`：`highlight_funny_hero_burst_ha`，点击后中间大 `哈`
- `bubble_echo`：`highlight_funny_bubble_hahaha`，右上固定对话框
- `edge_text`：`highlight_funny_text_haha`，边缘小 `哈哈`
- `edge_sparkle`：`highlight_funny_sparkle_diamond`
- `edge_dot`：`highlight_funny_sparkle_dot`

这套素材的当前结论：

- `highlight_funny_core_ha` 已验证和主大 `哈` 重复，最终删除，不再保留双层中心主字
- 默认态只保留中下按钮
- 点击后再出现中间大 `哈`、右上 `哈哈哈` 气泡和四周边缘素材
- 中间大 `哈` 与右上对话框固定位置，不参与随机分发
- 随机分发的只有边缘 `哈哈`、星点和圆点

当前 funny 的素材取舍原则：

- 粒子和点状装饰只做陪衬，密度要低于 `哈哈` 小字
- 底部边缘素材不要占中轴，否则会被大 `哈` 挡住
- 边缘分发优先沿左右外缘和上缘走，不往按钮正上方堆积
- funny 的热闹感主要靠更多 `哈` 字，而不是靠大量闪点

## 6.8 生成素材时的实际指导

生成式工具在这条链路里更适合做“资产母版”，不适合直接产出最终可运行组件。

优先生成这些内容：

- 单独的按钮本体
- 单独的中心主字/主图
- 单独的边缘小素材
- 单独的点击爆发素材

不建议让模型一次性生成：

- 带完整背景的大场景图
- 已经把按钮、主字、边缘粒子全部烘死在一起的整图
- 靠复杂背景光影才能成立的素材

实际提示词方向建议收敛为：

- 先描述“单个资产”的职责，而不是描述整个播放页
- 强调 `transparent background`
- 强调是 `mobile short-video overlay asset`
- 强调素材应靠近屏幕边缘使用，不遮挡中心主视频
- 对边缘素材额外强调：
  `small`, `scattered`, `echo`, `secondary`, `not centered`

如果后续继续补别的类型，也建议按这套顺序做：

1. 先出按钮本体
2. 再出中心主视觉
3. 再出 2 到 3 个边缘陪衬小素材
4. 最后补点击峰值爆发层

## 7. 五类高光的资产优先级

如果人力有限，不要五类平均做。

推荐优先级：

### 第一优先

- `reversal`
- `conflict`
- `sweet`

原因：

- 这三类最依赖真实质感
- 纯代码最难做像
- 最容易出现“像功能、不像成品”的问题

### 第二优先

- `funny`
- `feel_good`

原因：

- 这两类即使先用代码顶一版，也不会特别违和
- 但如果补了真实主字效和粒子，质感会明显提升

## 8. 生成式资产的执行方法

### 8.1 生成目标

生成式工具不负责一次生成完整可运行组件，而是负责生成：

- 主按钮视觉母版
- 主字 / 主图母版
- 边缘元素素材
- 峰值爆发参考层

### 8.1.1 当前阶段的现实约束

本项目原本也考虑过补齐更完整的工具层，例如：

- Figma 拆层与导出
- 更成熟的动效编辑工具
- 自动透明背景处理链路

但如果这些工具在当前阶段无法稳定接入，就不应继续把它们当成前置条件。

当前执行口径统一调整为：

- 不等待额外工具打通
- 直接用现有生成能力产出第一版资产母版
- 先把“能落地、能接入、能验证效果”的版本做出来
- 后续如果工具链补齐，再做更标准的分层和导出

也就是说，当前路线不是“先补工具，再做素材”，而是：

`先做素材参考 -> 先接入验证 -> 后补工具链升级`

### 8.2 生成原则

- 统一在短剧播放页语境内生成，不要带整部手机壳图
- 默认态和峰值态分开描述
- 强调透明背景可拆分思维
- 统一视觉风格，避免五类像来自五个产品

### 8.2.1 无额外工具前提下的执行方式

如果当前没有稳定可用的 Figma、Rive、After Effects 或自动拆层工具，则按以下方式推进：

1. 先生成一张高质量“峰值态参考图”
2. 再生成一张更克制的“默认触发态参考图”
3. 通过这两张图提取：
   - 按钮形态
   - 主字或主图风格
   - 颜色与辉光方向
   - 边缘元素类型
4. Android 端先不追求完整逐帧复刻，而是：
   - 用静态主图
   - 配合轻量放大、漂浮、淡入淡出
   - 组合出第一版效果

这种方案的核心是：

- 生图负责决定“长什么样”
- 代码负责决定“怎么出现和消失”

这样即使工具层不足，也不会卡住主线推进。

### 8.3 生成后的处理

生成后不要直接把整张图当成资产塞给客户端。

应做二次拆分：

- 提取主图
- 提取边缘元素
- 裁出按钮本体
- 记录颜色和光效特征

必要时再转成：

- `PNG`
- `WebP`
- `Lottie` 参考重做稿

### 8.3.1 当前阶段不强依赖自动拆层

在缺少稳定拆层工具的情况下，第一版不强求每张生成图都立即变成标准化透明分层资源。

可以先按下面的顺序处理：

- 第一步：把生成图作为视觉母版与实现参考
- 第二步：根据视觉母版提炼客户端需要的最小元素
- 第三步：优先接入最关键的主视觉和边缘元素
- 第四步：等效果验证通过后，再回头做资源精细化

这意味着当前阶段的“素材落地”允许先偏参考驱动，而不是一次性完成全部工业化整理。

## 9. 推荐的资产格式

第一版建议优先这三类格式：

### 9.1 `PNG`

适合：

- 主字主图
- 按钮本体
- 小图标元素

优点：

- 接入最快
- 调试方便

### 9.2 `WebP`

适合：

- 体积更小的静态透明图
- 轻量边缘素材

### 9.3 `Lottie` 或 `Rive`

适合：

- 需要明显形变的主视觉
- 翻牌、弹跳、柔光脉冲这类节奏明确的动效

第一版策略：

- 能先用静态图 + 代码编排的，就先不用复杂格式
- 只有主视觉动感不够时，再升级为 `Lottie / Rive`

## 10. Android 接入策略

Android 端不应直接依赖“大而全”的动画文件，而应按层接入：

- `TriggerButtonLayer`
- `MainVisualLayer`
- `PeakBurstLayer`
- `EdgeAtmosphereLayer`

运行时职责：

- Compose 决定位置和层级
- 点击触发峰值
- 控制淡入淡出、漂移、重复点击递进
- 负责避让字幕和人物核心区域

资产职责：

- 提供真正的视觉质感
- 提供字形、火焰、爱心、裂纹、粒子等不可替代的外观信息

## 12. 每类资产的最小交付包

第一版每一类高光不必追求很多资源，先满足最小可用包：

- 1 个触发按钮本体
- 1 个主视觉
- 1 个峰值爆发层
- 2 到 4 个边缘元素

只要这套最小包齐了，Android 端就能先做出比纯代码明显更像样的版本。

## 13. 哪些效果必须补真实素材

以下部分建议视为“必须补”：

- `reversal` 的卧槽 / WOC 主字、能量环、裂片和边缘回声层
- `conflict` 的火焰、余烬、火线
- `sweet` 的柔和爱心和暖光层

以下部分可先代码顶一版，再决定是否升级：

- `feel_good` 的边缘爽字和碎光
- `funny` 的部分上浮小字和轻粒子
