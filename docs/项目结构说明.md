## 目录职责

`android-app/`

- Android 客户端工程目录
- 当前已承载短剧列表、播放页、高光互动、历史回流、尾集分支等客户端逻辑

`server/`

- 本地服务端工程目录
- 当前承载内容管理、高光下发、互动统计、分支任务、AI 编排等能力

`admin-web/`

- 本地 Web 管理后台目录
- 当前承载高光管理、资源配置、互动数据查看、任务管理、演示数据初始化

`scripts/`

- 离线处理脚本目录
- 包含高光提取、文本结构化、分支生成辅助脚本等

`data/`

- 导出数据、seed 辅助数据与中间产物目录
- 服务端主运行库不在这里，而在 `server/data/`

`server/data/`

- 服务端运行数据库与测试数据库目录
- 当前主要包含 `app.db` 和 `test.app.db`

`assets/`

- 结构化静态资源目录
- 包含封面、剧集元数据、预生成分支结果、参考素材与特效资源
- 当前主要分为：
  - `assets/dramas/covers/`
  - `assets/dramas/episodes/`
  - `assets/dramas/branches/`
  - `assets/generated/fixed-branches/`
  - `assets/generated/manual-storyboard-tests/`
  - `assets/reference/branch-characters/`
  - `assets/effects/`

`videos/`

- 本地原始视频资源目录
- 不进入 Git
- 建议按 `videos/<drama_title>/第N集.mp4` 组织

`config/`

- 配置模板目录
- 存放 `.env.example`、资源扫描配置和本地部署模板

`docs/submission-assets/`

- 交付截图与录屏目录
- `screenshots/` 用于收纳提交版截图
- `videos/` 用于收纳最终录屏或演示视频

## 资源与字段规则

- 数据库存储资源相对路径 `path`
- 客户端和后台访问资源时由服务端统一映射为 `url`
- 固定分支的视频或旧资源路径仍可能落在 `assets/dramas/branches/`
- 固定分支当前主消费结果以 `assets/generated/fixed-branches/<episode_id>/` 下的 sidecar 和 manifest 为准
- 手工校准过的分镜测试图统一放在 `assets/generated/manual-storyboard-tests/`
- 剧封面建议放在 `assets/dramas/covers/<drama_id>.jpg`
