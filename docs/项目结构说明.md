# 项目结构说明

## 目录职责

`android-app/`

- Android 客户端工程目录
- 后续承载短剧列表、播放页、高光互动、尾集分支等客户端逻辑

`server/`

- 本地服务端工程目录
- 后续承载内容管理、高光下发、互动统计、分支任务、AI 编排等能力

`admin-web/`

- 本地 Web 管理后台目录
- 后续承载高光管理、资源配置、互动数据查看、任务管理、演示数据初始化

`scripts/`

- 离线处理脚本目录
- 包含高光提取、文本结构化、分支生成辅助脚本等

`data/`

- 本地数据库、种子数据、导出数据目录
- 建议至少包含 `app.db`、`seed/`、`exports/highlights/`、`exports/branches/`

`assets/`

- 结构化静态资源目录
- 包含封面、预生成分支资源、特效资源等
- 建议按 `assets/dramas/covers/`、`assets/dramas/branches/`、`assets/effects/` 组织

`videos/`

- 本地原始视频资源目录
- 不进入 Git
- 建议按 `videos/<drama_title>/第N集.mp4` 组织

`config/`

- 配置模板目录
- 存放 `.env.example`、资源扫描配置和本地部署模板

## 资源与字段规则

- 数据库存储资源相对路径 `path`
- 客户端和后台访问资源时由服务端统一映射为 `url`
- 固定分支视频建议放在 `assets/dramas/branches/<episode_id>/<option_id>.mp4`
- 剧封面建议放在 `assets/dramas/covers/<drama_id>.jpg`

## 当前说明
